# 分析证据收敛与稳定性设计

更新时间：2026-07-18（Asia/Shanghai）

## 目标

提高分析对交易决策的可用性，优先消除三类不可控误差：输入证据变化、Provider 自报置信度不可比、`unclear` 没有行动信息。该层不修改 judgement Prompt 和 evidence scoring 权重，也不把历史弱标签包装成收益概率。

## 输入冻结与严格 A/B

- `evidence_fingerprint` 对有序完整 packet（包括候选内容和 selected 状态）做 canonical JSON SHA-256。
- `prompt_fingerprint` 对实际保存并发送的完整 Prompt 做 SHA-256。
- `parent_run_id` 记录克隆来源；克隆只创建独立分析库草稿，不调用 Provider。
- 只有两个指纹都相同，才可把输出差异归因到 Provider/模型行为。候选或已选 Jaccard 只用于描述非严格对照，不能替代指纹。
- 旧行不强制回填；读取时从原始 JSON/Prompt 动态计算，避免迁移改写历史结果。

## 核心证据选择

候选上限继续为 40。默认核心包最多 8 条：先按既有相关度排序选取不同主题，再用剩余高分项补齐；最低 4 条和现有阈值、噪声/汇总规则保持不变。排序增加 `news_id` 作为最终 tie-break，消除相同分数和时间下的数据库顺序不确定性。

## 证据质量而非校准概率

本地质量分只使用：已选数量、本地相关度、催化是否引用已选消息、方向内部一致性、结构化行情涨跌与催化方向是否吻合。它不读取 `overall_confidence`，所以不同 Provider 自报 5% 或 99% 不会改变等级。

- A：证据扎实；B：可供决策；C：仅作观察；D：证据不足。
- 方向与行情冲突、内部同时多空或 judgement 为 `unclear` 时最高 C。
- 核心证据少于 2 条或多数催化无法映射到已选证据时最高 D。
- 页面必须明确该分数不是胜率、收益概率或交易信号。

不确定结果仍输出主假设、反证/冲突、最关键缺失证据、翻转条件和行动含义；C/D 或冲突状态明确要求等待独立证据或价格/成交量确认。

## 离线敏感性

Top-4/6/8 与 leave-one-out 沿用保存答案，仅切换证据可见集合并重算本地质量。该方法能发现结论是否依赖单条或边缘消息，但不能预测 Provider 在重新调用后会怎样改写答案。真实输出敏感性测试仍需未来显式授权的固定克隆 Provider 调用。

## 数据与外部边界

- 新增字段和克隆只写 `data/dashboard_analysis.sqlite3`。
- 稳定性列表和 CLI 使用 SQLite `mode=ro` / `query_only`。
- 不写业务历史库，不请求金十 REST，不触发 Telegram，不自动调用 Provider。
- 不自动投票、不平均 Provider 原始置信度、不覆盖历史答案。
